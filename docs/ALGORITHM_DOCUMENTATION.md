## 1. Общее описание программы

Программа предназначена для **объединения нескольких Excel-файлов** с данными об измерениях (расходы, давления, температуры) с разных узлов контроля и создания единого отчёта с цветовым форматированием.

### Что делает программа:

1. Пользователь выбирает несколько Excel-файлов, где в каждом файле хранятся данные с одного узла измерения
2. Программа читает специальный файл с правилами (`Правила названия столбцов.xlsx`), который описывает, как переименовать столбцы из каждого файла в единые стандартные имена
3. Все файлы объединяются в одну таблицу: строки — это временные метки, столбцы — разные параметры с разных узлов
4. Создаётся красивый Excel-отчёт с двумя листами:
   - **Лист "Данные"** — полная таблица с цветовой подсветкой нарушений
   - **Лист "Dashboard"** — сводный дашборд по дням с мини-графиками

### Технологии:

| Библиотека    | Для чего используется                          |
|---------------|------------------------------------------------|
| `tkinter`     | Создание графического интерфейса (окна, кнопки)|
| `pandas`      | Работа с табличными данными                    |
| `numpy`       | Математические вычисления                      |
| `openpyxl`    | Чтение файлов `.xlsx`                          |
| `xlrd`        | Чтение старых файлов `.xls`                    |
| `xlsxwriter`  | Запись Excel с форматированием и графиками     |

## 4. Вспомогательные функции (верхний уровень)

Эти функции определены вне класса — они являются «инструментами», которые используются классом.

---

### `resource_path(relative_path)`

**Строки:** 11–21

**Простыми словами:** Функция определяет, где находится файл, который нужен программе (например, файл правил или иконка). Это важно, потому что программа может запускаться по-разному: из папки с кодом, после установки через `pip` или из упакованного `.exe` файла.

**Как работает:**

```python
def resource_path(relative_path):
    try:
        # Если программа упакована в .exe через PyInstaller,
        # временные файлы хранятся в специальной папке _MEIPASS
        base_path = sys._MEIPASS
    except Exception:
        # Иначе берём папку, где лежит сам скрипт
        base_path = os.path.dirname(os.path.abspath(__file__))

    # Склеиваем базовый путь и имя файла
    path = os.path.join(base_path, relative_path)
    return path
```

**Пример использования:**
```python
rules_file = resource_path("Правила названия столбцов.xlsx")
# → "C:\Program Files\analytics_ui\Правила названия столбцов.xlsx"
```

---

### `get_column_letter(col_idx)`

**Строки:** 24–30

**Простыми словами:** Переводит **число** в **букву столбца Excel**. В Excel первый столбец — A, второй — B, ..., 26-й — Z, 27-й — AA и так далее. Эта функция нужна для построения формул условного форматирования.

**Как работает:**

Алгоритм похож на перевод числа из десятичной системы в 26-ричную (алфавит из 26 букв):

```
col_idx = 1  → "A"
col_idx = 26 → "Z"
col_idx = 27 → "AA"
col_idx = 28 → "AB"
```

**Пример использования:**
```python
letter = get_column_letter(3)  # → "C"
# Используется в формулах: "=AND(C4<10, C4<>0)"
```

## 5. Класс ExcelMerger: описание и методы

Класс `ExcelMerger` — это главный «мозг» программы. Он:
- Создаёт и управляет графическим интерфейсом
- Хранит все данные (список файлов, выбранные параметры, правила)
- Выполняет обработку данных

> **Что такое класс?** Это шаблон, который объединяет данные (переменные) и действия (функции-методы) в одну структуру. Все методы класса обращаются к его данным через `self`.

---

### 5.1 Инициализация `__init__`

**Строки:** 324–342

**Простыми словами:** Это первое, что выполняется при создании объекта программы. Здесь задаются начальные значения всех переменных и запускается создание интерфейса.

**Что происходит:**

| Действие | Результат |
|---|---|
| `self.root.title(...)` | Устанавливает заголовок окна |
| `self.root.geometry("1200x600")` | Задаёт размер окна: 1200×600 пикселей |
| `self.files = []` | Создаёт пустой список для хранения путей к файлам |
| `self.parameter_vars = {}` | Словарь для хранения состояния чекбоксов параметров |
| `self.node_vars = {}` | Словарь для хранения состояния чекбоксов узлов |
| `self._reload_rules()` | Загружает файл правил |
| `self.create_widgets()` | Строит графический интерфейс |

### `_reload_rules()`

**Строки:** 344–354

**Простыми словами:** Читает Excel-файл с правилами переименования столбцов и сохраняет его в переменную `self.rules_df`. Если файл не найден — создаёт пустую таблицу и пишет предупреждение в лог.

**Важно:** Этот метод вызывается не только при запуске, но и после **сохранения изменений в редакторе диапазонов** — чтобы программа работала с актуальными данными.

### 5.3 Работа с файлами

---

#### `add_files()`

**Строки:** 526–544

**Простыми словами:** Открывает диалоговое окно выбора файлов, позволяет пользователю выбрать один или несколько Excel-файлов. Затем обновляет все связанные элементы интерфейса.

**Алгоритм:**

```
1. Открыть диалог выбора файлов (только .xlsx и .xls)
2. Отфильтровать уже добавленные файлы
3. Если новых файлов нет — выйти
4. Для каждого нового файла:
   - Добавить путь в self.files
   - Показать имя файла в списке интерфейса
5. Обновить список узлов измерения
6. Обновить список параметров
7. Для каждого файла прочитать данные и обновить временной диапазон
```

---

#### `remove_file()`

**Строки:** 546–557

**Простыми словами:** Удаляет выбранный в списке файл. Если пользователь не выбрал ни одного файла в списке — ничего не происходит.

**Алгоритм:**

```
1. Узнать, какая строка выделена в списке
2. Удалить файл из self.files (по индексу)
3. Удалить строку из визуального списка
4. Обновить список узлов
5. Обновить список параметров
6. Сбросить временной диапазон
```

---

**Строки:** 628–633

**Простыми словами:** Читает Excel-файл и возвращает его содержимое как таблицу pandas (DataFrame). Поддерживает два формата:

| Формат | Библиотека | Когда используется |
|--------|-----------|-------------------|
| `.xlsx` | `openpyxl` | Новые файлы Excel (2007+) |
| `.xls`  | `xlrd`     | Старые файлы Excel (до 2003) |

#### `remove_empty_columns(df)`

**Строки:** 620–626

**Простыми словами:** Удаляет из таблицы столбцы, в которых **все** значения пустые (NaN). Это нужно для очистки данных от лишних пустых колонок.

```python
# Найти столбцы, где всё пусто
empty_cols = df.columns[df.isna().all()].tolist()

df = df.drop(columns=empty_cols)
```

### 5.4 Параметры и узлы измерения

---

#### `load_parameters()`

**Строки:** 488–514

**Простыми словами:** Читает из файла правил список уникальных параметров (5-й столбец) и создаёт для каждого чекбокс в центральной колонке интерфейса. По умолчанию все параметры отмечены галочкой.

**Алгоритм:**

```
1. Очистить старые чекбоксы
2. Если файл правил пустой — показать ошибку и выйти
3. Получить уникальные значения из 5-го столбца файла правил
4. Для каждого параметра:
   - Создать переменную BooleanVar (хранит True/False)
   - Создать чекбокс, связанный с этой переменной
   - По умолчанию установить True (выбран)
```

---

#### `select_all_parameters()` / `deselect_all_parameters()`

**Строки:** 516–524

**Простыми словами:** Устанавливают/снимают галочки со **всех** чекбоксов параметров одним нажатием кнопки.

---

#### `update_measurement_nodes()`

**Строки:** 569–618

**Простыми словами:** Анализирует имена загруженных файлов и находит, каким узлам измерения они соответствуют (по правилам из файла правил). Создаёт чекбоксы для найденных узлов.

**Алгоритм:**

```
1. Очистить старые чекбоксы узлов
2. Для каждого загруженного файла:
   a. Взять имя файла (без пути, в нижнем регистре)
   b. Пройтись по всем строкам файла правил
   c. Если имя файла совпадает с шаблоном в правиле:
      - Добавить название узла (4-й столбец) в множество узлов
3. Для каждого найденного узла:
   - Создать чекбокс (по умолчанию выбран)
```

> **Важно:** Сравнение имён файлов делается с нормализацией Unicode (`unicodedata.normalize('NFC', ...)`). Это нужно для корректного сравнения русских букв, которые в разных системах могут быть закодированы по-разному.

---

#### `select_all_nodes()` / `deselect_all_nodes()`

**Строки:** 559–567

Аналогичны функциям для параметров, но работают с чекбоксами узлов.

---

#### `get_rename_rules(filename)`

**Строки:** 635–672

**Простыми словами:** Для конкретного файла находит правила переименования столбцов. Возвращает словарь вида `{"старое_имя": "новое_имя"}`.

**Алгоритм:**

```
1. Взять имя файла в нижнем регистре
2. Получить список выбранных параметров
3. Пройтись по строкам файла правил
4. Для каждой строки проверить:
   - Совпадает ли шаблон имени файла с именем текущего файла?
   - Входит ли параметр в список выбранных?
5. Если оба условия выполнены → добавить в словарь переименований
6. Вернуть словарь
```

**Пример результата:**
```python
{
    "Q_газ_вход": "Расход_Узел1",
    "T_вход":     "Температура_Узел1"
}
```

### 5.7 Главная функция объединения `merge_files`

**Строки:** 839–1056

**Простыми словами:** Это главная функция программы — именно она выполняет всю обработку данных. Вызывается при нажатии кнопки "Объединить файлы".

**Подробный алгоритм:**

```
ШАГ 1: ПРОВЕРКИ

ШАГ 2: ПОСТРОЕНИЕ СЛОВАРЕЙ ИЗ ПРАВИЛ
──────────────────────────────────────
• param_to_columns — словарь {параметр: {столбец1, столбец2, ...}}

ШАГ 3: ОПРЕДЕЛЕНИЕ РАЗРЕШЁННЫХ СТОЛБЦОВ

• Собрать столбцы для всех выбранных параметров

ШАГ 4: ЧТЕНИЕ И ОБРАБОТКА КАЖДОГО ФАЙЛА

Для первого файла:
  • Сохранить столбец времени (первый столбец)
  • Удалить временной столбец из данных

Для остальных файлов:
  • Удалить временной столбец (если есть)

Для каждого файла:
  • Удалить пустые столбцы
  • Применить правила переименования
  • Оставить только разрешённые столбцы
  • Добавить в список dfs

ШАГ 5: ОБЪЕДИНЕНИЕ
────────────────────
• merged_df = pd.concat(dfs, axis=1)
  (объединение "горизонтально" — добавляем столбцы справа)
• Удалить дублирующиеся столбцы
• Вставить столбец времени в начало

ШАГ 6: ФИЛЬТРАЦИЯ ПО ВРЕМЕНИ

• Если задано начало — оставить строки с Время >= начало

• Удалить пустые столбцы
• Добавить столбцы со стрелками (↑↓) → add_arrow_columns()
```

**Простыми словами:** Делает лист "Данные" красивым и информативным. Добавляет трёхстрочные заголовки с объединёнными ячейками, применяет цветовое форматирование, устанавливает ширину столбцов.

**Алгоритм:**

```
ШАГ 1: СОЗДАНИЕ СТИЛЕЙ
─────────────────────────
• header_format — серый жирный заголовок (фон #D9D9D9)

• worksheet.hide_gridlines(2) — убирает серую сетку Excel

Из файла правил читается:

────────────────────────
```

## 7. Функция добавления стрелок `add_arrow_columns`

**Строки:** 261–320

**Простыми словами:** Рядом с каждым столбцом данных добавляет столбец-индикатор с символом `⚠`. В этот столбец записываются стрелки: ↓ если значение меньше нормы, ↑ если больше. Это позволяет быстро увидеть нарушения.

**Пример:**

```
Расход_Узел1 | Расход_Узел1 ⚠ | Температура | Температура ⚠
─────────────┼─────────────────┼─────────────┼───────────────
50.3         |                 | 18.5        |
8.1          | ↓               | 45.0        | ↑
120.5        | ↑               | 15.2        |
```

**Алгоритм:**

```
1. Создать словари:
   • param_limits — {столбец: {min: X, max: Y}}
   • param_to_node — {столбец: узел}

2. Для каждого столбца (из исходного списка):
   Если для него заданы min/max:
     а. Создать пустой столбец "ИмяСтолбца ⚠"
     б. Где значение < min (и != 0) → поставить "↓"
     в. Где значение > max (и != 0) → поставить "↑"

3. Переупорядочить столбцы:
   [данные] [⚠] [данные] [⚠] ...
   (столбец-индикатор всегда идёт сразу после столбца данных)

4. Вернуть обновлённый DataFrame и словарь param_to_node
```

> **Почему исключаются нули?** Нулевые значения обычно означают отсутствие данных (прибор не работал), а не реальное нарушение. Поэтому для нулей стрелки не ставятся.

## 8. Функция создания дашборда `create_dashboard_sheet`

**Строки:** 1059–1372

**Простыми словами:** Создаёт лист "Dashboard" — наглядную сводку по каждому узлу за каждый день. В ячейках — суммарный расход за день, цвет ячейки показывает статус, а поверх ячейки нарисован мини-график (спарклайн).

**Структура дашборда:**

```
┌──────────────┬──────────────────────┬───────────────┬──────────────┬──────────────┐
│ Узел (поз.)  │ Допустимый диапазон  │ Выход за диап │  01.01.2024  │  02.01.2024  │
├──────────────┼──────────────────────┼───────────────┼──────────────┼──────────────┤
│              │         Итого:       │ 5 сут.; 12 ч. │    450.32    │    423.10    │
├──────────────┼──────────────────────┼───────────────┼──────────────┼──────────────┤
│   Узел 1     │  (10 ... 100 тыс.м3) │  2 сут.; 5 ч. │ 🟢 125.44   │ 🔴 8.21     │
├──────────────┼──────────────────────┼───────────────┼──────────────┼──────────────┤
│   Узел 2     │  (5 ... 80 тыс.м3)   │  3 сут.; 7 ч. │ 🟢 88.30    │ 🟡 3.10     │
└──────────────┴──────────────────────┴───────────────┴──────────────┴──────────────┘

Цвета ячеек:
🟢 Зелёный — данные в норме
🔴 Красный — есть нарушения диапазона
🟡 Жёлтый  — расход малый (≤ 12 тыс. м3 за сутки)
⬜ Серый   — нет данных (все значения = 0)
```

**Алгоритм:**

```
ШАГ 1: ПОИСК СТОЛБЦОВ РАСХОДА
───────────────────────────────
• Пройтись по всем столбцам DataFrame
• Если параметр называется как "расход" (содержит слово "расход"):
  - Сохранить: узел, min, max, единицы, индекс столбца
• Если столбцов расхода нет → написать сообщение и выйти

ШАГ 2: ПОДГОТОВКА СПИСКА ДАТ
──────────────────────────────
• df['Date'] = pd.to_datetime(df['Время']).dt.date
  (из метки времени оставить только дату)
• unique_dates — уникальные даты, отсортированные по возрастанию
• Для каждой даты найти диапазон строк (start_row, end_row)

ШАГ 3: ЗАГОЛОВКИ
──────────────────
• Строка 0: "Узел", "Диапазон", "Статистика", [даты...]
• Строка 1: "Итого" (заполняется позже)
• Закрепить первые 2 строки и 3 столбца

ШАГ 4: ЗАПОЛНЕНИЕ МАТРИЦЫ
───────────────────────────
Для каждого узла и каждой даты:
  а. Взять значения из столбца расхода за эту дату
  б. Посчитать:
     • day_sum — сумма за день
     • min_violations — строки с значением < min (и != 0)
     • max_violations — строки с значением > max
     • day_violation_hours — количество часов с нарушениями

  в. Определить цвет ячейки:
     • Все нули               → серый (нет данных)
     • day_sum ≤ 12           → жёлтый (малый расход)
     • есть нарушения         → красный
     • всё в норме            → зелёный

  г. Записать сумму в ячейку с выбранным цветом фона

  д. Нарисовать спарклайн (мини-график):
     • Цвет линии зависит от цвета ячейки
     • Данные берутся из листа "Данные" за тот же день

ШАГ 5: СТАТИСТИКА
──────────────────
• Для каждого узла: записать "X сут.; Y ч." в столбец статистики
  Красный цвет, если есть хоть одно нарушение
• Строка "Итого": суммарная статистика по всем узлам

ШАГ 6: ИТОГОВАЯ СТРОКА
────────────────────────
• Для каждой даты: сумма расходов по всем узлам
• Общая статистика нарушений
```

## 9. Запуск программы: функции `setup_logging` и `main`

### `setup_logging()`

**Строки:** 1375–1386

**Простыми словами:** Настраивает систему журналирования — записи событий в файл-журнал. Это помогает диагностировать ошибки.

```
2024-01-15 10:30:45,123 - root - INFO - Обработка файла: data.xlsx
2024-01-15 10:30:46,456 - root - ERROR - Ошибка при сохранении файла: ...
```

---

### `main()`

**Строки:** 1389–1405

**Простыми словами:** Точка входа — с этой функции начинается работа программы.

**Алгоритм:**

```
1. Настроить журналирование
2. Создать главное окно Tkinter (root)
3. Создать объект ExcelMerger(root) — это строит весь интерфейс
4. Запустить цикл обработки событий root.mainloop()
   (программа ждёт действий пользователя — нажатий, кликов)
5. При критической ошибке:
   - Вывести сообщение в консоль
   - Напечатать полный стек вызовов
   - Ждать нажатия Enter (чтобы пользователь успел прочитать)
```

> **Что такое mainloop?** Это бесконечный цикл, который постоянно проверяет: не нажал ли пользователь кнопку? Не переместил ли мышь? Если да — вызывает нужный обработчик. Программа "живёт" внутри этого цикла, пока пользователь не закроет окно.

## 11. Словарь терминов

| Термин | Объяснение |
|--------|-----------|
| **DataFrame** | Таблица данных из библиотеки pandas. Похожа на таблицу Excel — строки и столбцы |
| **NaN** | "Not a Number" — специальное значение, означающее "пусто" или "нет данных" |
| **Tkinter** | Встроенная Python-библиотека для создания оконных приложений |

| **Спарклайн** | Маленький мини-график внутри одной ячейки Excel. Показывает тренд данных |
| **Условное форматирование** | Автоматическое изменение цвета ячейки в зависимости от её значения |

| **freeze_panes** | "Закрепить области" в Excel — строки и столбцы, которые остаются видимыми при прокрутке |
| **BooleanVar** | Переменная Tkinter типа True/False, которая "привязывается" к чекбоксу |
| **Toplevel** | Дополнительное окно Tkinter поверх главного (например, окно редактора диапазонов) |
